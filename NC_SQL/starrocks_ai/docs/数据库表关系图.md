# 用友NC数据库表关系图

## 概述
本文档详细描述了用友NC系统中各表之间的关联关系，按业务模块分类整理。

## 表分类

### 一、基础档案表（不需要pk_corp过滤）
- BD_INVBASDOC（存货档案）
- BD_STORDOC（仓库档案）
- BD_CUBASDOC（客商档案）
- BD_PSNDOC（人员档案）
- BD_DEPTDOC（部门档案）
- BD_AREACL（地区档案）
- BD_BILLTYPE（单据类型）
- BD_BUSITYPE（业务类型）
- BD_DEFDOC（自定义档案）
- BD_CUSTADDR（客户地址）

### 二、管理档案表（需要pk_corp过滤）
- BD_CUMANDOC（客商管理档案）
- BD_INVMANDOC（存货管理档案）
- BD_PAYTERM（收付款协议）
- SO_CUCREDIT（客商额度信用表）

### 三、业务单据表（需要pk_corp过滤）
- SO_SALE（销售订单主表）
- SO_SALEINVOICE（销售发票主表）
- SO_SALEINVOICE_B（销售发票附表）

- TO_BILL_B（调拨单附表）

### 四、业务数据表（需要pk_corp过滤）
- IC_ONHANDNUM（现存量表）
- IA_GENERALLEDGER（存货总账）

### 五、视图
- V_ZLW_CUST（客户视图）

## 核心业务关系图


### 1. 客户信息关系链

```
客户完整信息查询路径：

BD_CUMANDOC (客商管理档案) [主表]
    ├─ pk_cubasdoc → BD_CUBASDOC (客商档案) [基本信息]
    ├─ pk_payterm → BD_PAYTERM (收付款协议) [账期信息]
    ├─ pk_resppsn1 → BD_PSNDOC (人员档案) [业务员]
    ├─ pk_respdept1 → BD_DEPTDOC (部门档案) [部门]
    │   └─ pk_fathedept → BD_DEPTDOC (部门档案) [上级部门-自关联]
    └─ custflag: '0'/'2'=客户, '1'=供应商

BD_CUBASDOC (客商档案)
    ├─ pk_cubasdoc → SO_CUCREDIT (信用额度表) [可欠款信用]
    │   └─ climittypeid = '0001A110000000002EXU'
    ├─ pk_cubasdoc → SO_CUCREDIT (信用额度表) [临时信用]
    │   └─ climittypeid = '0001A110000000002EXV'
    └─ pk_areacl → BD_AREACL (地区档案) [省市信息]
        └─ pk_fatherarea → BD_AREACL (地区档案) [自关联：省→市]

快捷视图：
V_ZLW_CUST (客户视图) - 包含客户、部门、业务员、省市等完整信息
```

**关键字段：**
- BD_CUMANDOC.pk_cumandoc：客商管理主键（业务关联用）
- BD_CUBASDOC.pk_cubasdoc：客商档案主键（基础信息用）
- BD_CUMANDOC.innerctldays：信用天数
- BD_PAYTERM.termid：账期天数（'01'表示0天）
- SO_CUCREDIT.nlimitmny：信用额度
- SO_CUCREDIT.vcredittypename：额度属性（默认'2'）

### 2. 存货信息关系链


- BD_INVBASDOC.pk_invbasdoc = BD_INVMANDOC.pk_invbasdoc

- BD_INVBASDOC.pk_invbasdoc = IC_ONHANDNUM.cinvbasid
- BD_INVMANDOC.pk_invmandoc = IC_ONHANDNUM.cinventoryid

**成本价格来源：**
- IA_GENERALLEDGER.nabprice：结存单价（推荐）

```
应收应付单据：

ARAP_DJFB (应收应付单据分表)
    ├─ pk_djfb: 单据分表主键
    ├─ hbbm → BD_CUBASDOC (客商档案)

    ├─ cinventoryid → BD_INVMANDOC (存货管理)
    ├─ djdl: 单据大类

账期超期计算：
超期天数 = DATEDIFF(CURDATE(), DATE_ADD(billdate, INTERVAL CAST(termid AS INT) DAY))
```

## 常用关联模式速查表

### 模式1：客户完整信息
```sql
FROM BD_CUMANDOC cuman
INNER JOIN BD_CUBASDOC cumbase ON cumbase.pk_cubasdoc = cuman.pk_cubasdoc
LEFT JOIN BD_PAYTERM term ON cuman.pk_payterm = term.pk_payterm
LEFT JOIN BD_PSNDOC psn ON psn.pk_psndoc = cuman.pk_resppsn1
LEFT JOIN BD_DEPTDOC dept ON cuman.pk_respdept1 = dept.pk_deptdoc
LEFT JOIN SO_CUCREDIT redit ON cumbase.pk_cubasdoc = redit.pk_cubasdoc 
    AND redit.climittypeid = '0001A110000000002EXU'
```

### 模式2：存货完整信息
```sql
FROM BD_INVBASDOC inv
INNER JOIN BD_INVMANDOC invman ON inv.pk_invbasdoc = invman.pk_invbasdoc
LEFT JOIN IA_GENERALLEDGER gl ON invman.pk_invmandoc = gl.cinventoryid
```

### 模式3：库存查询
```sql
FROM IC_ONHANDNUM oh
INNER JOIN BD_INVBASDOC inv ON oh.cinvbasid = inv.pk_invbasdoc
INNER JOIN BD_STORDOC stor ON oh.cwarehouseid = stor.pk_stordoc
LEFT JOIN IA_GENERALLEDGER gl ON oh.cinventoryid = gl.cinventoryid 
    AND oh.ccalbodyid = gl.crdcenterid
```

### 模式4：销售毛利分析
```sql
FROM SO_SQUAREDETAIL sq
INNER JOIN BD_INVMANDOC invman ON sq.cinventoryid = invman.pk_invmandoc
INNER JOIN BD_INVBASDOC invbas ON invman.pk_invbasdoc = invbas.pk_invbasdoc
INNER JOIN BD_DEPTDOC dept ON sq.cdeptid = dept.pk_deptdoc
INNER JOIN BD_PSNDOC psn ON sq.cemployeeid = psn.pk_psndoc
LEFT JOIN BD_CUMANDOC cuman ON sq.ccustomerid = cuman.pk_cumandoc
LEFT JOIN BD_CUBASDOC cubas ON cuman.pk_cubasdoc = cubas.pk_cubasdoc
LEFT JOIN BD_BUSITYPE btp ON sq.cbiztype = btp.pk_busitype
```

### 模式5：销售发票明细
```sql
FROM SO_SALEINVOICE ii
INNER JOIN SO_SALEINVOICE_B ib ON ib.csaleid = ii.csaleid
INNER JOIN BD_INVBASDOC inv ON ib.cinvbasdocid = inv.pk_invbasdoc
LEFT JOIN SO_SALE od ON ib.coriginalbillcode = od.vreceiptcode
LEFT JOIN BD_CUMANDOC cum ON ii.creceiptcustomerid = cum.pk_cumandoc
LEFT JOIN BD_CUBASDOC cub ON cum.pk_cubasdoc = cub.pk_cubasdoc
LEFT JOIN BD_PSNDOC psn ON ii.cemployeeid = psn.pk_psndoc
```

### 模式6：应收账龄
```sql
FROM ARAP_DJFB fb
LEFT JOIN BD_PAYTERM ter ON fb.sfkxyh = ter.pk_payterm
LEFT JOIN BD_CUBASDOC cub ON fb.hbbm = cub.pk_cubasdoc
LEFT JOIN V_ZLW_CUST cu ON cub.pk_cubasdoc = cu.pk_cubasdoc
WHERE fb.djdl = 'ys' AND fb.isverifyfinished = 'N'
```

### 模式7：采购入库
```sql
FROM IC_GENERAL_H h
INNER JOIN IC_GENERAL_B b ON h.cgeneralhid = b.cgeneralhid
INNER JOIN BD_INVBASDOC inv ON b.cinvbasid = inv.pk_invbasdoc
INNER JOIN BD_BILLTYPE bt ON h.cbilltypecode = bt.pk_billtypecode
LEFT JOIN BD_STORDOC stor ON h.cwarehouseid = stor.pk_stordoc
LEFT JOIN SM_USER us ON h.coperatorid = us.cuserid
WHERE h.fbillflag >= 3
```

### 模式8：调拨单
```sql
FROM TO_BILL zb
INNER JOIN TO_BILL_B db ON zb.cbillid = db.cbillid
INNER JOIN BD_STORDOC rc ON db.cinwhid = rc.pk_stordoc
INNER JOIN BD_STORDOC cc ON db.coutwhid = cc.pk_stordoc
INNER JOIN BD_INVBASDOC ch ON db.cinvbasid = ch.pk_invbasdoc
LEFT JOIN SM_USER us ON zb.coperatorid = us.cuserid
```

## 特殊关联说明

### 1. 地区信息（省市）
```sql
-- BD_AREACL 自关联
SELECT 
    cl.pk_areacl,
    c.areaclname AS 省,
    cl.areaclname AS 市
FROM BD_AREACL c
INNER JOIN BD_AREACL cl ON c.pk_areacl = cl.pk_fatherarea
```

### 2. 部门层级（上下级）
```sql
-- BD_DEPTDOC 自关联
SELECT 
    dept.deptname AS 部门,
    fdept.deptname AS 上级部门
FROM BD_DEPTDOC dept
LEFT JOIN BD_DEPTDOC fdept ON dept.pk_fathedept = fdept.pk_deptdoc
```

### 3. 信用额度（两种类型）
```sql
-- SO_CUCREDIT 关联两次
LEFT JOIN SO_CUCREDIT redit 
    ON cumbase.pk_cubasdoc = redit.pk_cubasdoc
    AND redit.climittypeid = '0001A110000000002EXU'  -- 可欠款信用
LEFT JOIN SO_CUCREDIT redits 
    ON cumbase.pk_cubasdoc = redits.pk_cubasdoc
    AND redits.climittypeid = '0001A110000000002EXV'  -- 临时信用
```

### 4. 存货总账（需要双字段关联）
```sql
-- IC_ONHANDNUM → IA_GENERALLEDGER
LEFT JOIN IA_GENERALLEDGER gl 
    ON oh.cinventoryid = gl.cinventoryid 
    AND oh.ccalbodyid = gl.crdcenterid  -- 必须同时匹配库存组织
```

### 5. 销售订单号关联
```sql
-- 通过单据号字符串关联
LEFT JOIN SO_SALE od 
    ON ib.coriginalbillcode = od.vreceiptcode  -- 字符串匹配
```

## 关联注意事项

### 1. pk_corp 过滤规则
- **需要过滤**：单据表、管理档案表、业务数据表
- **不需要过滤**：基础档案表、系统表、视图

### 2. dr 过滤规则
- **所有表**都需要添加 `dr = 0` 过滤条件

### 3. 关联类型选择
- **INNER JOIN**：必须有数据的关联（如主附表）
- **LEFT JOIN**：可能没有数据的关联（如业务员、部门等）

### 4. 特殊字段处理
- **termid = '01'**：表示0天账期，需要转换
- **custflag IN ('0', '2')**：客户，'1'表示供应商
- **blargessflag = 'Y'**：赠品，'N'表示非赠品
- **def3 = '0001C11000000012TJ3X'**：商桥客户

### 5. 视图使用建议
- **V_ZLW_CUST**：推荐用于快速获取客户完整信息
- 视图已包含常用关联，无需再次关联基础表
